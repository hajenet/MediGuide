# MediGuide Marketing Plan

## 목표

MediGuide를 글로벌 간호 학생과 예비 의료인을 위한 다국어 의학용어/기초간호 학습 앱으로 발전시킨다. 1주일 안에 배포 가능한 MVP는 웹/PWA로 출시하고, 이후 사용자 반응을 확인한 뒤 Google Play와 App Store 진입을 검토한다.

## 핵심 포지셔닝

MediGuide는 "짧은 시간에 반복 학습하는 다국어 간호/의학 퀴즈 앱"으로 포지셔닝한다.

초기 강점:

- 설치 없이 바로 실행 가능한 웹 앱
- 의학 약어와 핵심 용어 중심의 빠른 학습
- 오답 복습과 반복 학습에 적합한 구조
- 영어, 한국어를 시작으로 다국어 확장 가능
- 국가별 시험명에 직접 종속되지 않는 범용 학습 콘텐츠

피해야 할 포지셔닝:

- 공식 국가시험 문제 제공 앱
- 특정 시험 기관이 인증한 앱
- 전문 의료 판단을 제공하는 앱
- 병원 실무 의사결정 도구

## 시장 조사 요약

간호 학습 앱 시장에는 이미 여러 경쟁 앱이 있다. 경쟁 앱들은 대체로 문제은행, 해설, 플래시카드, 모의고사, 통계, 광고 또는 인앱결제를 조합한다.

확인한 경쟁/참고 사례:

- CozyNCLEX Prep: NCLEX 스타일 문제, CAT 시뮬레이션, spaced repetition, 진도 통계, 커스텀 플래시카드 제공
- NursePrep: 주제별 간호 문제, 즉시 해설, 광고와 인앱결제, 로그인 없는 구조
- Pocket Prep Nursing School: 대규모 문제은행, 모의고사, 약점 분석, 여러 간호 관련 시험 지원
- Nursing Terminology Quiz: 유료 앱, 오프라인 용어/플래시카드/퀴즈/통계 제공
- Google Play의 Nursing Exam 앱: 광고 포함, 50만+ 다운로드, 모의고사/노트/리더보드 제공
- 한국의 틈틈봇 간호조무사 앱: 잠금화면 자동 학습, 광고 포함, 짧은 반복 학습 컨셉

시사점:

- 단순 퀴즈만으로는 차별화가 약하다.
- 글로벌 시장에서는 NCLEX, HESI, TEAS, 간호학교 시험, 의학용어 학습 수요가 크다.
- 다국어 지원은 차별화 포인트가 될 수 있지만, 번역 품질과 의학 정확도가 핵심 리스크다.
- 1주일 MVP에서는 대규모 문제은행보다 "잘 정리된 소량 콘텐츠 + 반복 학습 경험"이 현실적이다.

## 수익화 아이디어

### 1. 광고 기반 무료 모델

구성:

- 무료 퀴즈 제공
- 결과 화면 또는 세션 종료 후 광고
- 보상형 광고를 보고 추가 문제/오답 복습/힌트 제공

장점:

- 진입 장벽이 낮다.
- 학생 타깃에 적합하다.
- 초기 사용자 확보에 유리하다.

단점:

- 웹 MVP에서는 광고 심사와 수익 발생까지 시간이 걸릴 수 있다.
- 의료/교육 콘텐츠에서 과한 광고는 신뢰도를 떨어뜨린다.
- 사용자 수가 작으면 수익이 작다.

1주일 MVP 적합도: 중간

### 2. 유료 프리미엄

구성:

- 무료: 기본 의학용어/약어 퀴즈
- 유료: 전체 문제은행, 오답 복습, 카테고리별 시험, 통계, 오프라인 PWA 기능

가격 예시:

- 월 구독: USD 2.99-4.99
- 연 구독: USD 19.99-29.99
- 평생 이용: USD 19.99-49.99

장점:

- 광고보다 교육 앱 신뢰도에 유리하다.
- 콘텐츠 품질이 높아질수록 수익화가 쉽다.

단점:

- 결제 구현과 스토어 심사가 필요하다.
- 글로벌 가격 현지화가 필요하다.
- 초기 콘텐츠가 적으면 전환율이 낮다.

1주일 MVP 적합도: 낮음

### 3. 문제팩 판매

구성:

- 무료 기본팩
- 유료 문제팩
  - Medical Abbreviations Pack
  - Cardiovascular System Pack
  - Respiratory System Pack
  - Pharmacology Basics Pack
  - NCLEX Terminology Pack

장점:

- 콘텐츠 단위로 가치 제안이 명확하다.
- 구독 거부감이 있는 사용자에게 적합하다.

단점:

- 결제 시스템이 필요하다.
- 문제 품질과 양이 핵심이다.

1주일 MVP 적합도: 낮음

### 4. B2B/학교 라이선스

구성:

- 간호학과, 학원, 온라인 교육자에게 클래스용 링크 또는 문제 세트 제공
- 교수가 직접 문제를 추가하는 버전 제공

장점:

- 단가가 높다.
- 광고 없이 운영 가능하다.

단점:

- 영업과 신뢰 확보가 필요하다.
- 관리자 기능과 학습 리포트가 필요하다.

1주일 MVP 적합도: 낮음

## 다국어 전략

### 데이터 구조

문제 데이터는 언어별 텍스트를 포함하는 구조가 좋다.

```js
{
  id: "cardio-001",
  category: "cardiovascular",
  terms: ["CV", "Cardiovascular"],
  question: {
    en: "Which abbreviation refers to cardiovascular medicine?",
    ko: "순환기내과를 지칭하는 약어는?"
  },
  options: ["CV", "GS", "NS", "OS"],
  answer: "CV",
  explanation: {
    en: "CV stands for Cardiovascular and is used for heart and blood vessel related care.",
    ko: "CV는 Cardiovascular의 약어로 심장과 혈관 관련 진료를 뜻합니다."
  }
}
```

### 번역 운영 원칙

- 영어를 기준 언어로 둔다.
- 한국어는 첫 번째 로컬라이즈 언어로 둔다.
- 의학 용어는 번역보다 원어 병기를 우선한다.
- 자동 번역만으로 배포하지 않는다.
- 각 언어별 검수자를 두는 것이 이상적이다.
- 국가시험명, 기관명, 상표명은 공식 명칭과 사용 가능 범위를 확인한다.

- nursing flashcards
- NCLEX terminology practice
- anatomy and physiology quiz
- medical English for nurses
- 간호 의학용어 퀴즈
- 간호 약어 공부

## 이슈 및 리스크

### 1. 이미지 라이선스

해부학 이미지, 심장 구조, ECG 파형, 장기 그림을 가져올 때 라이선스가 핵심 이슈다.

주의할 점:

- Google 이미지 검색 결과를 그대로 사용하면 안 된다.
- 병원, 대학, 교재, 논문 이미지에는 저작권이 있을 수 있다.
- Wikimedia Commons도 파일별 라이선스가 다르므로 개별 확인이 필요하다.
- OpenStax Anatomy and Physiology는 라이선스가 버전별로 다르므로 페이지별 조건을 확인해야 한다.
- CC BY 이미지는 출처 표기가 필요하다.
- CC BY-NC는 상업 앱에서 사용하기 어렵다.
- 생성형 AI 이미지도 학습/사용 조건과 상업 이용 가능 여부를 확인해야 한다.

MVP 권장:

- 이미지 기능은 1주일 MVP에서 제외한다.
- 필요하면 직접 제작한 단순 SVG 또는 라이선스가 명확한 공개 자료만 사용한다.
- 모든 이미지에 `source`, `license`, `author`, `url` 메타데이터를 기록한다.

### 2. 의료 정보 책임

앱은 학습 보조 도구이며 의료 조언이 아니라는 문구가 필요하다.

권장 문구:

```text
MediGuide is for educational purposes only and does not provide medical advice, diagnosis, or treatment.
```

### 3. 시험명/상표권

NCLEX, HESI, TEAS 등은 시험명 또는 상표일 수 있다. 앱 설명에서 "official", "certified", "licensed"처럼 오해를 줄 수 있는 표현을 피한다.

권장 표현:

- "NCLEX-style terminology practice"
- "For nursing students preparing for school exams and terminology review"

피해야 할 표현:

- "Official NCLEX app"
- "NCLEX certified questions"
- "Real exam questions"

### 4. 문제 저작권

기출문제, 교재 문제, 유료 문제은행을 그대로 복제하면 안 된다.

MVP 권장:

- 자체 제작 문항만 사용한다.
- 출처 기반 설명이 필요하면 공개 라이선스 자료를 참고하고 출처를 남긴다.
- 문제 데이터에 `sourceNote` 필드를 둘 수 있다.

### 5. 다국어 품질

의학용어 오역은 신뢰도에 큰 타격을 준다.

대응:

- 영어 원문을 기준으로 관리한다.
- 번역 검수 상태를 데이터에 기록한다.
- 검수 전 언어는 "beta"로 표시한다.

## 결론

1주일 안에 배포하려면 "글로벌 다국어 웹/PWA 퀴즈 MVP"로 범위를 줄이는 것이 현실적이다. 이미지 기반 해부학 게임, 심장 구조 퍼즐, 결제, 광고는 매력적이지만 초기 배포 범위에서는 제외하는 편이 낫다.

가장 먼저 검증해야 할 가설은 다음이다.

1. 글로벌 간호 학생이 짧은 의학용어 퀴즈를 반복해서 사용할까?
2. 영어와 모국어를 함께 보여주는 방식이 학습에 도움이 될까?
3. 사용자가 오답 복습과 카테고리 학습을 위해 재방문할까?

이 세 가지가 확인되면 문제팩, 광고, 유료 기능, 앱스토어 배포 순서로 확장한다.
